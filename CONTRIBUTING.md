# 贡献指南

感谢你愿意改进慢慢来。这个项目的目标是做一个不制造压力、但能帮助用户重新行动的计划卡牌应用。

## 开发流程

1. Fork 仓库并创建功能分支。
2. 保持改动聚焦，一次提交解决一个清晰问题。
3. 提交前运行：

```powershell
.\gradlew.bat test
.\gradlew.bat assembleDebug
```

4. 发起 Pull Request，并说明改动内容、验证方式和影响范围。

## 代码风格

- Kotlin 和 Compose 代码优先跟随现有写法。
- UI 改动需要同时考虑深色主题、不同牌面样式和小屏手机。
- 任务提醒相关改动要确认完成、删除、恢复任务后的提醒行为。
- 数据模型变更需要考虑 Room 迁移。

## 设计原则

- 鼓励用户继续行动，而不是制造羞耻感。
- 首页优先保留任务卡牌体验，不退回传统待办列表。
- 牌面、主题和动效应该服务于任务管理，而不是喧宾夺主。

## 问题反馈

提交 issue 时请尽量包含：

- 设备型号和 Android 版本。
- 应用版本或 commit id。
- 复现步骤。
- 期望行为和实际行为。
- 如有 UI 问题，请附截图。
